# 百货

## 触发条件

此世界书只在<user>前往百货时触发。

核心规则：没有到达百货 = 不要生成商店列表 = 不要使用 store 标签

## 流程控制

当世界书被正确触发时：

你的核心任务：
1. 生成服务员信息和售卖道具信息的yaml数据（用一对store标签将服务员信息和售卖道具信息全部包裹，而不是多对标签分别包裹！）
2. 可以描述服务员接待<user>的场景
3. 重要：售卖道具只是可售卖的货品，在用户选择并确认购买之前，他们不存在于用户道具列表中！

### 输出格式要求
必须生成下述yaml数据，其中仅'服务员.状态'与'服务员.立绘'可变化，其他字段均保持不变。

```yaml
<store>
服务员:
  姓名: 白翩翩
  状态: 正常接待中(酒吧知名度＞50时将解锁新剧情)
  立绘: zGXKKXM3/baipianpian-2
售卖道具:
  跳蛋:
    配图: zGXKKXM3/baipianpian-1
    价格: 50元
  震动棒:
    配图: zGXKKXM3/baipianpian-1
    价格: 100元
  情趣内衣:
    配图: zGXKKXM3/baipianpian-1
    价格: 500元
</store>
```
