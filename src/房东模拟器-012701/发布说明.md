# 项目发布指南

## GitHub 发布流程

### 初始化仓库（首次发布）
```bash
cd d:\AI\MySpecialFolder\Code\tavern_room
git init
git add .
git commit -m "初始提交"
git remote add origin https://github.com/NoteBene/tavern_room.git
git branch -M main
git push -u origin main
```

### 后续更新发布
```bash
cd d:\AI\MySpecialFolder\Code\tavern_room
git add .
git commit -m "更新说明"
git push origin main
```

### 处理合并冲突
当远程仓库已有内容时，可能需要处理合并冲突：
```bash
# 拉取远程更改并允许合并不相关历史
git pull origin main --allow-unrelated-histories

# 解决冲突文件后提交
git add <冲突文件路径>
git commit -m "解决合并冲突"
git push -u origin main
```

## jsDelivr CDN 发布

### 触发CDN缓存更新
每次GitHub更新后，需要刷新jsDelivr缓存以获取最新内容：

**PowerShell命令：**
```powershell
Invoke-WebRequest -Uri "https://purge.jsdelivr.net/gh/NoteBene/tavern_room@main/" -Method GET
```

**curl命令（Git Bash）：**
```bash
curl -X GET "https://purge.jsdelivr.net/gh/NoteBene/tavern_room@main/"
```

### CDN资源访问URL
- **基础URL**: `https://cdn.jsdelivr.net/gh/NoteBene/tavern_room@main/`
- **图片资源**: `https://cdn.jsdelivr.net/gh/NoteBene/tavern_room@main/img/文件名.jpg`
- **HTML文件**: `https://cdn.jsdelivr.net/gh/NoteBene/tavern_room@main/dist/房东模拟器-012701/界面/状态栏/index.html`

## 注意事项
1. GitHub仓库必须设置为**公开**，jsDelivr才能访问
2. 文件路径中的中文字符会被自动编码，确保URL正确性
3. CDN缓存刷新后通常需要1-5分钟生效
4. 建议在重要更新后手动触发CDN刷新，确保用户获取最新版本
